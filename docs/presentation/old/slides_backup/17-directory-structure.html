ディレクトリ構造と実行フロー

プロジェクト構造

Cb/
├── src/           # インタプリタ実装
│   ├── lexer.c    # 字句解析
│   ├── parser.c   # 構文解析
│   ├── ast.c      # AST構築
│   ├── eval.c     # 評価エンジン
│   └── ...
├── stdlib/        # 標準ライブラリ（Cbで記述）
│   └── std/
│       ├── vector.cb
│       ├── queue.cb
│       └── time.cb
├── tests/         # テストスイート
└── docs/          # ドキュメント

実行フロー

1. ソースコード読み込み
   └─ main.cb

2. 字句解析（Lexer）
   └─ トークン列生成

3. 構文解析（Parser）
   └─ AST構築

4. 型チェック
   └─ 型安全性の検証

5. 実行（Interpreter）
   └─ AST木を再帰的に評価
   
6. 標準ライブラリのロード
   └─ stdlib/ 配下のモジュール
