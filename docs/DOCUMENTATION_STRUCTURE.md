## メンテナンスガイドライン

1. **機能を実装したとき**:
   - 設計ドキュメントを`todo/`から`archive/releases/vX.Y.Z/`に移動
   - `IMPLEMENTATION_STATUS.md`を更新
   - `release_notes/vX.Y.Z.md`にリリースノートを作成

2. **機能を計画するとき**:
   - 設計ドキュメントを`todo/`または`todo/vX.Y.Z/`に作成
   - `IMPLEMENTATION_STATUS.md`で参照

3. **バージョン番号付け**:
   - 主要機能: マイナーバージョンをインクリメント（v0.X.0）
   - バグ修正/小機能: パッチバージョンをインクリメント（v0.X.Y）
   - 破壊的変更: TBD（安定版はv1.0.0+を使用予定）
