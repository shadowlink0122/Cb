## 🎯 ドキュメント管理方針

### 新機能の開発フロー

1. **計画段階**: `docs/todo/`に設計書を作成
2. **開発中**: `docs/features/`に実装状況を記録
3. **実装完了**: リリースノートを作成
4. **アーカイブ**: `docs/archive/features/vX.X.X_*/`に移動

### アーカイブ条件

以下の条件を満たした機能はアーカイブされます:
- ✅ 完全に実装され、全テストが合格
- ✅ リリースノートに記載
- ✅ 実装ドキュメントが完成
- ✅ 次期バージョンがリリース済み

---

## 📝 ドキュメント作成ガイドライン

新しい機能のドキュメントを作成する場合は、以下の構成を推奨します:

1. **概要** - 機能の簡潔な説明
2. **機能仕様** - 詳細な仕様と構文
3. **実装された機能** - 何ができるか
4. **テスト結果** - 動作確認結果
5. **使用例** - 実践的なコードサンプル
6. **実装の詳細** - アーキテクチャと技術的特徴
7. **既知の制限事項** - 制限や注意点
8. **テスト実行方法** - テストの実行手順
9. **結論** - まとめ
10. **関連ファイル** - 関連するファイルへのリンク
