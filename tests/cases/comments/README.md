# コメント機能のテスト

このディレクトリには、Cb言語のコメント機能に関するテストが含まれています。

## テストファイル

### 1. test_line_comment.cb
行コメント `//` の基本的な動作をテストします。

**テスト内容:**
- 行の先頭のコメント
- 行末のコメント
- 連続した複数行のコメント
- コメントアウトされたコード

**期待される動作:**
- コメント部分は無視される
- コメント以外のコードは正常に実行される

### 2. test_block_comment.cb
ブロックコメント `/* */` の基本的な動作をテストします。

**テスト内容:**
- 単行のブロックコメント
- 式の途中のブロックコメント
- 複数行のブロックコメント
- コメントアウトされた複数行のコード

**期待される動作:**
- `/*` から `*/` までがコメントとして無視される
- コメント以外のコードは正常に実行される

### 3. test_mixed_comments.cb
行コメントとブロックコメントの混在をテストします。

**テスト内容:**
- 同じ行に行コメントとブロックコメントが存在する場合
- 式の中に複数のブロックコメントが存在する場合
- ブロックコメントと行コメントが連続する場合

**期待される動作:**
- 両方の種類のコメントが正しく処理される
- コメント以外のコードは正常に実行される

### 4. test_comment_in_string.cb
文字列リテラル内のコメント記号をテストします。

**テスト内容:**
- 文字列内の `//` はコメントとして扱われない
- 文字列内の `/* */` はコメントとして扱われない
- 文字列の外のコメントは正常に処理される

**期待される動作:**
- 文字列リテラル内のコメント記号は文字列の一部として扱われる
- 文字列の外のコメントは正しく無視される

### 5. test_comment_edge_cases.cb
コメントのエッジケースをテストします。

**テスト内容:**
- 空のブロックコメント `/**/`
- スペースのみのブロックコメント
- 演算子の前後のコメント
- 括弧の前後のコメント
- 関数呼び出しの引数内のコメント

**期待される動作:**
- 様々な位置のコメントが正しく処理される
- コメントによって式の評価が妨げられない

## テストの実行方法

個別のテストを実行:
```bash
./main tests/cases/comments/test_line_comment.cb
./main tests/cases/comments/test_block_comment.cb
./main tests/cases/comments/test_mixed_comments.cb
./main tests/cases/comments/test_comment_in_string.cb
./main tests/cases/comments/test_comment_edge_cases.cb
```

統合テストスイートで実行:
```bash
make integration-test
```

## 注意事項

### サポートされている機能
- 行コメント `//`
- ブロックコメント `/* */`
- 文字列リテラル内のコメント記号の適切な処理

### サポートされていない機能
- ネストされたブロックコメント（例: `/* /* */ */`）

ネストされたブロックコメントは一般的なC系言語の仕様に従い、サポートされていません。
